功能权限设计本质是功能点与角色之间的一个矩阵表！但如果把所有功能点与所有角色一一对应，往往都有上万个配置！工作量太大！
所以要总结出一定规律与原则，加速这种配置！
角色的总结主要有三个维度：总行/一级分行/二级分行；各业务条线(主要是法律/合规/风险,其他部门)；五种人(领导,部门经理,经办人,普通员工,系统管理员)！！如果将这三个维度进行迪卡尔组合,一共是:3*4*5=60,即标准的应该有60种角色

功能点的总结一般有这几个维度：
是哪个业务模块的？比如：合规,法律,风险,
是业务功能,系统功能,还是基础数据功能？
是维护/查询/统计/流程处理？
我新的角色设计思路是:
1.总体原则上分为五种角色,公司领导,部门经理,经办员,一般员工,系统管理员，它们的区分原则是:
公司领导 ―― 只查看统计分析,一眼清,一键通导出等功能,保证他们进入系统时非常简便操作,这些人原则上不做任何录入或提交操作,或者说是在系统中不要干活的!
部门经理 ―― 也叫中层干部,这些人一般主要是负责工作流的审批与提交工作,同时有较高的数据查询权限(主要是本部门),它与领导的区别是他要在系统干活的!与经办员区别是他们干的活少些,主要是处理工作流,而不负责初始数据录入!!
经办员 ―― 就是具体干活的,尤其是初始数据录入与提交,比如制度维护,体系文件维护,风险点评估,风险事件/报告录入等,他们录入后一般提交给部门经理去审核!!他们的功能点权限最多,换句话说就是他们才是在系统中干活最多的!
一般员工 ―― 就是普通员工,只有一些查询的功能权限,而且数据权限也非常小,只限于本部门/本条线等
系统管理员 ―― 维护机构,角色,公告等系统级数据的人,同时具有较高的数据权限,可以查看本分行/全行数据,这种人最特殊,在行政岗位上他们是普通员工,但在系统权限上等同于部门经理甚至领导!!

这里面有变化的就是“经办员”，就是说有可能制度维护的“经办员”与文件或风险维护的“经办员”是不一样的!! 所以可以分为“制度经办人”,“合规经办人”,“风险经办人”,即细化成3个!! 理论上可以细化更多!但大部分客户其实有些职能都是合在一起的!!


角色这样划分还带来的一个好处就是，直接与系统功能点对应上了,即系统功能点有什么制度维护，文件维护，都分配给这些经办员了..


2.在总分行模式下,必须区分总行角色与分行角色,这样就会角色就会乘以2,共10个!!为什么只到分行而不到支行呢? 这是因为客户本身的许多管理职能都只下放到分行层级,也只有分行有相应的对口管理部门!!


其中功能权限设计是:
一般员工有内外规查询,文件查询,岗位说明书查询等所有非条线功能的查询功能,同时可能还有制度评价等对所有员工开放的功能!! 但没有检查,事件,报告等查询功能!!

制度经办人负责制度维护,内规映射,等功能！！ 但总行与分行的数据权限有区别

合规经办人负责体系文件维护,合规检查维护,合规事件录入,合规报告填报,等条线功能点的维护与录入功能!! 但总行与分行的数据权限有区别

风险经办人负责风险识别与评估,根据实际情况，也可以填报合规事情或报告!!

部门经理负责处理工作流外，还有具有风险点查询,风险矩阵,合规事件查询,合规报告查询等

系统管理员具有所有功能!!!但分行管理员的数据权限限于本分行的!!

需要特别指出的是,
我们在系统实施过程中,对于工作流处理,尽可能的强烈建议用户直接从合规部发起! 而不是让所有一般部门都参与工作流!! 然后让一般部门发起工作流!!!
换句话说就是,就是让合规部之前的流程在系统外处理,比如使用邮件,excel填报,然后由合规部在系统中填报流程!!

这样做的目的是，这会大大减低系统工作流的使用人数,即将系统工作流处理控制在合规条线,则系统培训与售后支持的工作量都会大大降低!!!
而且实际情况也完全能做到这一点,比如合规事情，合规报告，流程文件调整申请等，都可以简化成直接由合规部发起!!!


对于制度审查,法律审查,有时没办法，必须由一般部门发起!!!那在实施过程中也建议分行的流程可以由系统外先报送到合规部,然后由合规部再从系统中发起流程!! 如果客户实在不同意再说...

总的来说,系统中的工作流有四种情况:

1.总行合规条线发起报送
比如：总行合规经办人(王小蒙) -> 总行合规部门经理(刘大脑袋) -> 总行领导(王大拿)
2.总行非合规条线发起报送
比如: 总行计财部一般员工(王霞) -> 总行计财部部门经理(王老七) -> 总行合规经办人(王小蒙) -> 总行合规部门经理(刘大脑袋) -> 总行领导(王大拿)
3.分行合规条线发起报送
比如：分行合规经办人(李大国) ->分行合规部部门经理(赵四) -> 分行领导(刘能) -> 总行合规经办人(王小蒙) -> 总行合规部门经理(刘大脑袋) -> 总行领导(王大拿)
4.分行非合规条线发起报送
比如:分行计财部一般员工(陈艳南) ->分行计财部部门经理(李秋歌) -> 分行合规经办人(李大国) ->分行合规部部门经理(赵四) -> 分行领导(刘能) -> 总行合规经办人(王小蒙) -> 总行合规部门经理(刘大脑袋) -> 总行领导(王大拿)
