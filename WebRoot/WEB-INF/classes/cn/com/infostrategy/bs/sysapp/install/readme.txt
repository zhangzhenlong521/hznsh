【UI页面的主要原则】
1.主页面尽可能简单,最好就用一个单列表! 而且预先查出数据! 然后通过点击上方的按钮来进行下一步操作!! 这是屡试不爽的重要原则与经验!!!
2.在编辑界面的对象大多数应该是主表对象,比如[外规明细维护][外规解读][外规...],在查询界面大多是将主子表拉平一起显示!!!
3.查询界面如果是左边是树,右边是列表,通过选择树来过滤列表,那么在列表上面不需要查询面板! 且大多情况下,选择树结点自动将树结点的所有子结点拼成in条件去过滤! 这样更形象!

【按钮的处理】
1.如果是风格模板或一个自定义的面板中的按钮,首选是直接代码写按钮,如果该按钮可能在项目中会进行个性化权限配置,则在菜单参数中进行配置! 当然也可以使用注册按钮,这样配置灵活性是解决了,但迁移麻烦了!
2.所以还有一种机制就是虽然在代码中写的按钮,但进行右键配置时,先查询一下数据库中有没有,如果有,则执行数据库中的权限过滤! 如果没有,则提示是否拷贝到数据库中进行配置,就像元原模板的拷贝机制一样! 但这样一来就需要一个唯一性编码来判别该按钮! 如何保证在代码中某个按钮的唯一性呢??
